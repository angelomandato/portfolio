# PHP Applications and Contributions by Angelo Mandato

Since 2000 Angelo Mandato has developed a number of PHP applications and made numerous contributions to the PHP community.

Angelo's journey with PHP began in fall of 2000 when he was developing C++ and Perl CGI applications at United McGill Corporation. Specifically Angelo was tasked with developing a web based application to manage and saving of company documents including time sheets, expense reports, internal request documents, and proposals. Development with C++ was slow and cumbersome, and Perl CGI scripts were difficult to maintain. Angelo evaluated PHP 4.0 for its ease of use, rapid development capabilities, and strong support for web applications.



## PHP at United McGill Corporation

 ### Document Center

 The Document Center web application was a Zoho like internal application initially built with C++ CGI, Xitami web server, and a FoxPro database to store document information. PDF FDF (Forms Data Format) form fields were used to allow users to fill out documents and submit them to the application. The application provided features such as document listing per user, editing and saving, search functionality, and an administrative interface for managing users, documents, and document access.

 The Document Center application was rewritten in PHP 4 using the Apache web server php module (rather than CGI) and utilized MySQL databases to store document information. The application provided features such as document listing, search functionality, and an administrative interface for managing documents. Later versions incorporated the ability to digitally sign documents, print to PDF, and

 ### Qmail Email Administration Server

 In 2001 Angelo developed a web based email administration server for the Qmail mail server. The application was developed in PHP and utilized a MySQL database to store email account information. The application provided features such as email account management, aliasing, and email forwarding. There was also a section that connected the accounts to the SMTP server (MIMESweeper) for managing the releasing of SPAM messages.

### Document Sharing Application

 In 2004 Angelo architected and oversaw the development of a document file sharing server that allowed employees to upload large documents for sharing in email. Primarily for AutoCAD drawings that were over 2MB in size, the file sharing application allowed engineers at the comiapony to share as well as receive large documents that otherwise would require disk media to send or receive. At this time email servers could not handle attachments over 1 or 2MB in size.

## PHP at Blubrry Podcasting
 
  Angelo developed and architected a number of applications and services for Blubrry Podcasting, including:
  
  * Podcasting Statistics - A podcast statistics application and service that provided detailed analytics for podcast episodes.
  * Generator - A podcast directory website that allowed users to subscribe to podcasts
  * Generator Mobile - A mobile-only website of the Generator platform. (Early years of mobile devices it was common to host a separate website for mobile devices only)
  * API - A REST api for all of Blubrry's services that supported both XML and JSON results.
  * Blubrry Publisher - A web application to allow podcast publishing customers a central interface for creating and publishing podcasts.
  * iCrawler - A podcast scraping engine that would find new podcasts and automatically add them to the Blubrry Podcast Directory.
  * Commerce - A commerce and billing system, initially developed to support PayPal subscriptions that expanded to support direct credit card payments via PayPal payments platform and later incorporated BrainTree payments processing.
  * Surveys - A unique survey system that allowed web visitors to fill out a simple survey to support podcast statistics data.
  * Subscribe - A unique subscription web application that powered both subscribebyemail.com and subscribeonandroid.com.
  * Exchange - An internal web application that facilitated advertising campaigns and creating ad-buys for clients.
